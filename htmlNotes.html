HTML

Hypertext Markup Language

Used for standard document design in websites

Notes

  • Add Tags - say why
  • Add html lang attribute to your header - sets the language of the content on the page
  • Fallowing heading structure h1 > h2 > h3 is good for Accessibility
  • Can drag and drop HTML file into browser and it will show content
  • Elements without closing tags can be "closed" by ending it in /
  • width is an attribute that can be added outside of the style attribute - not valid HTML
  • div is short for divider
  • alt + shft + F organizes code indents
  • HTML Document Tree
  • Src-set helps render different images per browser size
  • input has type attributes checkbox, color, date, text, password, time, file ect
  • input has a placeholder attribute
  • Head holds metadata - styles, titles, scripts
  • link rel="stylesheet" href="styles.css" - link to external css file - goes in head
  • Metadata controls what is seen when a user shares the link
  • Head holds the tags you see on the page
  • !doctype html specifies html version (html 5)
  • dl, description list, used as a contianer for a list of terms, encloses a list of groups of terms, commonly used to implement a glossory or display metadata (as key-value pairs)
  • Why write Alt Text
    • Makes site more accessible
    • Fallback incase image doesn't load
    • Helps site rank higher on google
  • Rules to writing alt text
    • don't use "image of"
    • Be concise (less than 125 characters) - a lot of screen readers will cut off after 125 characters
    • How would you describe it over the phone
  • ctrl + d = highlights all similar text, changes appropriate text

Definitions

ol
ordered list
ul
unordered list
li
list item
dl
description list
the parent element that encloses the list of terms and descriptions
dt
description term
the child element that specifies a term in a description list, must be used inside dl
dd
description details
the child element that provides the description, definition, or value for the proceding (dt) term in a dl
Web Standards
Guidlines and practices for the creation and delivery of content on the www.
These standards are crucial for ensuring web content is accessible, reliable, and compatable, across different devices and platforms.
The primary organization responsible for web standards is the world wide web consortium (w3c).
