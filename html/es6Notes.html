    Notes

    • Template Literals
      • `${word1} ${word2}`
      • `${num1 + num2}`
      • `${word1} ${word2}` Will show on multiple lines.
    • Destructuring Objects
      • const obj = { key1: 'value1' key2: 'value2' key3: { key31: 'value31' } } const { key1, key2, key3: { key31 } } = obj key1 key2 key31
      • Allows you to grab objects without referencing the object.
    • Destructuring Arrays
      • let [value1, value2, value3] = [ 'value1', 'value2', 'value3' ] value1 value2 value3
    • Object Literal
      • function functionName(param1, param2) { const newObj = { param1, param2 } }
    • For Of Loop
      • let arr = [1, 2, 3] let total = 0 for (const num of arr) { total += num }
      • let str = 'hello' for (const letter of str) { console.log(letter) }
      • const arr = [ { name: 'name1', age: 1 }, { name: 'name2', age: 2 }, { name: 'name3', age: 3 } ] for (const { name, age } of arr) { console.log(name, age) }
    • Spread Operator
      • const arr1 = [1, 2, 3] const arr2 = [4, 5, 6] const arr3 = [...arr1, ...arr2]
      • Creates a new copy of the array.
      • const obj1 = { key1: 'value1' } const obj2 = { key2: 'value2' } const obj3 = { ...obj1, ...obj2 }
    • The Rest Operator
      • function sum(...nums) { return nums.reduce((total, num) => total + num) } sum(1, 2, 3, 4, 5)
    • Arrow Functions
      • const add = (num1, num2) => num1 + num2 If you only have one line you can return on the same line.
      • const add = (num1, num2) => { return num1 + num2 }
      • const add = num => num + 1 If you only have one param you can remove the parenthasis
    • Default Params
      • function add(num1, num2 = 1) { return num1 + num2 }
      • Used for when you don't always have an argument for the parameter with the default.
    • includes()
      • const arr = [1, 2, 3] arr.includes(2)
      • Not supported by internet explorer.
      • expects an exact match.
    • Let and Const
      • variable hoisting issues ith var is solved by let and const, var hoists to the function scope level. if (false) { var name = 'Rae' } would return null
      • Let is a strictor version of var that uses block scope. if (false) { let name = 'Rae' } would return an undefined error, as the let doesn't exist unless it's true.
      • Const stands for constant. const name = 'Rae' name = 'Rae2' Cannot be reassigned. Would return an error as name is read-only.
      • Const handles arrays differently. const arr = [1, 2, 3] arr.push(4) Would work fine where as const arr = [1, 2, 3] arr = [4, 5, 6] would result in a syntax error as arr is read-only.
      • Const handles objects differently. const obj = { key1: 'value1' } obj.key2 = 'value2' Would work fine.
    • Import and Export
      • Allows us to follow solid principles, do dependancy injection, and more object oriented programming.
      • Allows code to be more modular in nature, and thus more organized.
      • // file1.js export const name = 'Rae' export const age = 30 // file2.js import { name, age } from './file1.js'
    • padStart() and padEnd()
      • Allows you to add to the start/end of a string.
      • let example = 'example' example.padEnd(10, 'a') would result in exampleaaa
      • You can pass in just a number let example = 'example' example.padStart(10) would return example with 3 empty characters at the beginning
      • let example = 'example' example.padEnd(1) would return example, as example is longer than one character.
    • Classes
      • class Person { constructor(name, age = 50) { this.name = name this.age = age } greet() { return `Hello, my name is ${this.name} and I am ${this.age} years old.` } } const person1 = new Person('Rae', 25) person1.greet() would return Hello, my name is Rae and I am 25 years old.
      • You can easily change the values person1.name = 'Fadumo' person1.age = 30 person1.greet() Would return Hello, my name is Fadumo and I am 30 years old.
      • class Person { constructor(name, age = 50) { this.name = name this.age = age } get metaData() { return `Name: ${this.name}, Age: ${this.age}` } } const person1 = new Person('Rae', 25) person1.metaData would return Name: Rae, Age: 25.
      • You can also inherit constructors and extend them class generation extends Person { constructor(name, age, generation) { super(name, age) this.generation = generation } metaData() { return `${this.name} is ${this.age} years old and from the ${this.generation} generation.` } } let person1 = new Person('Rae', 25, 'genz') person1.metaData() would return Rae is 25 years old and from the genz generation.
      • Be careful! It isn't always best to use this.
      • classes need constructors.
    • Trailing Commas
      • Debatable
      • functions can now have trailing commas.
      • function example(param1,) { const example = { key1: 'value1', key2: 'value2', } }
    • Promises
      • Usually used on network requests (fetching data from an API), or in an asynchronous function.
      • Promises are classes.
      • const promise = new Promise((resolve, reject) => { setTimeout(() => { if (!error) { resolve('success') } else { reject('error') } }, 2000) }) promise.then(successs => console.log(success)).catch((error) => console.log(error))
    • Fetch
      • Allows you to not be dependant on jquery.
      • fetch("https://jsonplaceholder.typicode.com/posts") .then(response => response.json()) .then(data => console.log(data))
      • response.json() maps out the body.
      • The default is a GET method.
      • To post fetch("https://jsonplaceholder.typicode.com/posts", { method: 'POST', body: JSON.stringify({ title: 'foo', body: 'bar', userId: 1 }) })
    • Async and Await
      • async function example() { const promise = new Promise((resolve, reject) => { setTimeout(() => { resolve('success') }, 2000) }) const result = await promise console.log(result) } example()
      • This will ensure your function waits for the promise.
      • async function example() { const response = await fetch("https://jsonplaceholder.typicode.com/posts") const data = await response.json() console.log(data) } example()
    • Sets
      • const exampleSet = new Set([1,1,1,1,1,1,2]) exampleSet.add(5) exampleSet.delete(1) exampleSet.has(1) exampleSet.clear()
      • A set is itterable
