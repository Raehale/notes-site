    Notes

    • Schemas
      • Schemas, pantrysoft-name, tables, find table, right click table, Select Rows
      • Schemas, pantrysoft-name, tables, find table, select table with lightning bolt icon
      • When table is selected information about the table/field will be in the block at the bottom left
      • Also shows indexes for tables
      • -bd-a is archive table
      • Select statements come back in the grid below. You can filter using the "filter rows" input field.
      • double click on the schema to focus on it (make it the target schema). The schema name is in the information box if it is selected.
    • Workbench
      • change password: server, users an priveledges, select user account, add new password, select apply in the bottom right.
      • Nothing is deleted on the first click you have to select apply.
      • If workbench is not responding, close it down and open it.
      • import dump command: open sql script, make sure dump is NEW. Make sure you delete all dumps when you no longer need the data.
    • Output
      • To view a whole row select it, right click, copy the row, paste it in the editor.
      • when on a table, if a field is too long to read, right click, open value in editor, text
    • Editor
      • limit is at top as dropdown when table loaded
      • Editor is where you run your queries.
      • You can save scripts (used for dumps usually).
      • Lightning bolt executes command
      • You can have multiple commands in an editor. Make sure they all end in a semicolon. It will run the first, than the second.
      • If you have multiple commands in an editory, you can highlight one and select the lightning bolt to run only that one.
      • Use ctrl + f for find/find and replace
    • SQL
      • select * FROM 'schema'.table; means select all from the table on schema
      • where name like 'rae' will show all the rows with the name as rae.
    • PS Facts
      • Agencies are optional. Agencies are parents over orgs, used as data warehouses. Consolidates different data, agency can run reports for all orgs under it. Not many people use it.
      • Cron, short for chronometor (time). runs jobs on a specific timer (npm) NPM JS Crons. PS has two cron, a nightly and hourly cron.
