    • Design Roles
      • User Researcher
        • Understand user behaviors and needs.
        • Gather and analyze user data.
        • Conduyct surveys, interviews, and usability testing
      • UX Designer
        • Responsible for the overall feel of the product.
        • Focus on how users will interact with the product.
        • Use wireframing tools like Balsamiq or prototyping tools like Figma or Adobe XD
      • UI Designer
        • Focus on designing the user interface of a site.
        • Create detailed mockups developers use to build the final product.
      • Padding sweet spot is 1.3em-1.9em
      • Line height should be around 1.5
      • Think of every element as having a series of rows and columns.
      • Add equal whitespace in container.
      • WCAG 2.5 Contrast Guidlines (minimum AA) say the visual presentation of text and images of text has a contrast ration of at least 4.5:1, except for large text which should have a contrast ratio of at least 3:1.
      • WCAG 2.5 Contrast Guidlines (enhanced AAA) say the visual presentation of text and images of text has a contrast ration of at least 7:1, except for large text which should have a contrast ratio of at least 4.5:1.
      • The size of every UI element must be carefully considered .
    • Typography - Good typography require the understanding of other fundamentals, along with a few special considerations.
    • Typography fundamentals
      • Font Choices - no more than two choices, idealy one
      • Visual - a way to establish the order of importance in terms of typography
      • Font size (scale)
      • Alignment
      • Letter spacing and line height
      • font styles (weight, italics, etc...)
      • Color and contrast
    • Colors
      • Color - The first UI design fundamental that shapes a user's experience is color.
      • Green - wealth, nature, growth
      • Black - luxury, sophistication, elegance
      • Certain colors mean certain things to certain cultures, be aware of your target audience.
      • Avoid using a bunch of colors in your UI Design
      • light up border bottom on inputs on focus.
    • Visual Hierarchy
      • Every element on a user interface has a level of importance. Some elements are more imporatn than others. Visual hierarchy is how we establish this importance.
      • White space, contrast, can be used to display a hierarchy.
      • For when theres a bubble with content on a page, go slightly lighter or slightly darker than the backgroud.
    • Montserrat is a good standard font to use.
    • Layout
      • Keep white space
      • monochromatic is an option for colors
      • Use fundamentals to determine layout
      • Using a "random" color, that has good contrast, to emphasize words/buttons.
    • Dark Mode
      • Not just inversing colors.
      • Avoid absolute black, esp with white text on it, as it causes eye strain.
    • When using a background image use colors from the image in the content to tie them together.
    • Sometimes you might need to darken/lighten your background to help with contrast.
    • When using a background image be aware of any high contrast issue areas there might be.
    • Borders/underlines are old school and add clutter.
    • White space is primarily established through padding and margin.
    • When thinking about mobility, consider how difficult it will be to select things.
    • Smaller buttons on mobile should be rectangle or have a border for clickability.

    Design Patterns
    Commonly used solutions to common problems in software design.
    Dark Mode
    Dark mode is a color scheme that uses light-colored text, icons, and graphical elements on a dark background.
    Card Layouts
    Card layouts are a design pattern that groups information into cards that can be easily rearranged and reorganized.
    Modal Dialogs
    Modal dialogs are a type of dialog that appears on top of the main content and requires the user to interact with it before they can continue.
    Shopping Carts
    Shopping carts are a design pattern that allows users to add items to a cart and then proceed to checkout.
    Breadcrumbs
    Breadcrumbs are a design pattern that shows the user's current location within a website or application.
    User Testing
    User testing includes A/B testing, heatmaps, and user behavior analytics.
    White Space
    The empty space between the elements in your user interface.
    Alignment
    Alignment is the process of ensuring that every element is positioned correctly in relation to other elements.
    Contrast
    Contrast is being in a strikingly different state from something else.
