Basics

  • Can drag and drop HTML file into browser and it will show content
  • Elements without closing tags can be "closed" by ending it in /
  • width is an attribute that can be added outside of the style attribute - not valid HTML
  • div is short for divider
  • alt + shft + F organizes code indents
  • HTML Document Tree
  • Src-set helps render different images per browser size
  • input has type attributes checkbox, color, date, text, password, time, file ect
  • input has a placeholder attribute
  • Head holds metadata - styles, titles, scripts
  • link rel="stylesheet" href="styles.css" - link to external css file - goes in head
  • Metadata controls what is seen when a user shares the link
  • Head holds the tags you see on the page
  • !doctype html specifies html version (html 5)
  • dl, description list, used as a contianer for a list of terms, encloses a list of groups of terms, commonly used to implement a glossory or display metadata (as key-value pairs)
  • Why write Alt Text
    • Makes site more accessible
    • Fallback incase image doesn't load
    • Helps site rank higher on google
  • Rules to writing alt text
    • don't use "image of"
    • Be concise (< 125 characters) - a lot of screen readers will cut off after 125 characters

CSS

Cascading style sheets

Used for aesthetic design

  • Colors
  • Backgrounds

Basics

  • font-weight: lighter, normal, or bold
  • display: inline; stacks items beside eachother horizontally
  • display: block; stacks items on top of eachother vertically, each item takes up full width
  • browser assumes body tag wants margin around
  • margin will collapse into eachother, largest margin will take presidence
  • center an element: display: block; set width, set margin left/right to auto
  • Use line-height instead of height for input fields - more browsers will honor it
  • Flexbox is god.
    • display-content: center, end, start, space-around, space-between
    • gap: px
    • only affects direct children (if you don't want something effected, change it to be within divs).
    • Divs are dispaly block by default
    • Don't use pure black or pure white because it will block/show all light emitted from the screen, making it difficult for your viewers.
    • background-size: cover (background is the width of the container), (google)
    • buttons don't inherit font-family (use font-family: inherit;)
    • the span tag is display: inline;
    • Use id for unique elements that will only be used once
    • Use classes for elements that will be reused
    • .webp is similar to .gif, but is more compressed
    • text-shadow: horizontal-line vertical-line blur color;

HTML

ol

ul

dl
description list
the parent element that encloses the list of terms and descriptions
dt
description term
the child element that specifies a term in a description list, must be used inside dl
dd
description details
the child element that provides the description, definition, or value for the proceding (dt) term in a dl

Relative Path Syntax
an address that points to the current directory or project folder
./ --> start from the same directory the current file is in - more explicit - visual reminder your referencing a file within the project
../ --> start from the parent directory of (the directory above) the current file
/ --> start from the project's root directory

General

Version Control with Git
Creats "save points" (commits) in your code that you can revert back to if needed.
Manages changes by "merging" multiple different versions (branches) of the same code base.
Generally allows you to be less worried about making mistakes or having your code conflict with other people working on the same project.
Github
A cloud based SaaS product, now owned by Microsoft.
Used for sharing your code with others.
Assumes the usage of the Git Version Control System
Has many Git-based capabilities built into it. (commiting, merging, branches, ect)
